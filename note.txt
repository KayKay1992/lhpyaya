====================================================
  FEATURE: Admin Can Delete a Registrant
====================================================

OVERVIEW
--------
Admins can now remove individual registrants from an event directly
from the ViewRegistered page. A "Remove" button appears on every row
of the registrants table. Clicking it asks for confirmation, then
permanently deletes the record from the database and removes the row
from the screen without needing to reload the page.

----------------------------------------------------
BACKEND CHANGES
----------------------------------------------------

1. controllers/registrationController.js
   - Added a new async function: deleteRegistrant
   - It receives the registration's unique ID from the URL parameter
     (:registrationId)
   - Uses Mongoose's findByIdAndDelete() to locate and remove the record
   - Returns 200 with a success message if found and deleted
   - Returns 404 if no matching registration was found
   - Returns 500 on unexpected server errors
   - Exported alongside the existing two functions

   Key code logic:
     const registration = await Registration.findByIdAndDelete(req.params.registrationId);
     if (!registration) return res.status(404).json({ message: "Registrant not found" });
     res.status(200).json({ message: "Registrant deleted" });

2. routes/registrationRoutes.js
   - Imported deleteRegistrant from the controller
   - Added a new DELETE route:
       DELETE /api/registrations/:registrationId
   - Protected with both `protect` (JWT check) and `adminOnly` (role check)
     middlewares, so only logged-in admins can call it
   - The :registrationId in the URL is the MongoDB _id of the registration
     document, NOT the event ID

----------------------------------------------------
FRONTEND CHANGES
----------------------------------------------------

3. pages/ViewRegistered.jsx
   - Added Trash2 icon import from lucide-react
   - Added deletingId state (null when idle, holds the _id of the
     registration currently being deleted — prevents double-clicks)
   - Added handleDelete(id) async function:
       * Shows window.confirm() before proceeding
       * Calls DELETE /api/registrations/:id via the axios instance
       * On success: filters the deleted entry out of the registrants
         state array (instant UI update, no re-fetch needed)
       * Shows a toast success or error message
       * Resets deletingId to null when done
   - Changed the table grid from grid-cols-4 to grid-cols-5 to add an
     Actions column on the right
   - Each row now has a "Remove" button:
       * Shows the Trash2 icon + "Remove" label
       * While deleting: shows "Removing..." and is disabled
       * Styled in red tones to signal a destructive action

----------------------------------------------------
HOW THE FULL FLOW WORKS
----------------------------------------------------

1. Admin navigates to the Registrants page for an event
2. The table lists all registrants with Name, Email, Phone, Date, and
   a Remove button per row
3. Admin clicks "Remove" on a row
4. Browser shows: "Remove this registrant?" — admin clicks OK
5. Frontend sends:  DELETE /api/registrations/<registrationId>
   with the JWT token in the Authorization header
6. Backend middleware verifies the token and confirms admin role
7. Controller deletes the document from MongoDB
8. Frontend receives success → filters that row out of state
9. The row disappears instantly. Toast shows "Registrant removed"

----------------------------------------------------
SECURITY NOTES
----------------------------------------------------
- The DELETE endpoint is protected by JWT authentication (protect
  middleware) — unauthenticated users cannot call it
- The adminOnly middleware ensures regular users (role: "user") get a
  403 Forbidden response even if they have a valid token
- The registration ID used in the URL is a MongoDB ObjectId — it is
  not guessable, and it can only be obtained by first viewing the
  registrants list (which is also admin-only)
