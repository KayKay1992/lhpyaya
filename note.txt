FEATURE: Hover Tooltips on Admin Registrant Table
===================================================

How it was done:
-----------------
The ViewRegistered.jsx page shows a table of registered participants.
On narrow screens, cells are truncated with Tailwind's `truncate` class,
meaning long emails or phone numbers are cut off with "...".

To let the admin see the full value without extra popups or libraries,
we added the native HTML `title` attribute to every `<span>` that holds
a truncatable value. When you hover over a cell, the browser automatically
shows the full text in a small tooltip bubble — no JavaScript or extra
component needed.

Files changed:
- frontend/src/pages/ViewRegistered.jsx

Cells updated (both desktop grid and mobile grid):
  - Name     → title={r.name}
  - Email    → title={r.email}
  - Phone    → title={r.phone || undefined}   (undefined hides empty tooltip)
  - Referral → title={r.referral || undefined}

Using `|| undefined` on optional fields means the tooltip only appears
when the cell actually has a value — no blank tooltip box on empty cells.

===================================================
FEATURE: Event Add-Ons Page + Registration Form Link
===================================================

User request:
  Add a link/button on the registration page (after the Register button)
  that says "Do you want a certificate of participation or speak with our
  keynote speaker one on one? Click here." This should open a new page
  where the participant can book the speaker session and/or pay for a
  certificate — just like the prompts on the success screen.

How it was done:
-----------------

1. New page — frontend/src/pages/EventAddons.jsx
   A standalone public page at the route /event-extras.
   It reads three query parameters from the URL:
     - eventId   (passed from the registration form)
     - title     (the event name, URL-encoded)
     - name      (the participant's name, pre-fills destination forms)

   The page shows two bold, coloured action cards side by side:

   CERTIFICATE CARD (amber theme):
   - Icon: Award (lucide-react), amber badge
   - Shows the fee: ₦2,000
   - Button navigates to /certificate-payment with the same query params
   - Participant then sees payment details, fills certificate name, uploads evidence

   SPEAKER BOOKING CARD (blue theme):
   - Icon: Mic2 (lucide-react), blue badge
   - Shows the fee: ₦20,000
   - Button navigates to /speaker-booking with the same query params
   - Participant fills phone, uploads payment evidence

   Bottom of the page has a "No thanks, go back to events" text link → /

2. Route registered — frontend/src/App.jsx
   Added:
     import EventAddons from "./pages/EventAddons";
     <Route path="/event-extras" element={<EventAddons />} />
   (public route, no auth required)

3. Link in registration form — frontend/src/pages/RegisterEvent.jsx
   Inside the white form card, immediately below the "Confirm Registration"
   submit button, added a small section separated by a top border:

     "Want a certificate of participation or a one-on-one session
      with our keynote speaker?"
     [Award icon] [Mic2 icon] Click here to book / pay →

   The link uses React Router's <Link> and builds the URL dynamically:
     /event-extras?eventId={eventId}&title={eventTitle}&name={encodeURIComponent(form.name)}

   This means the name typed into the form so far is passed along as a
   query param so the destination pages can pre-fill the participant's name.
   If the user hasn't typed a name yet, the field simply starts empty.

   The link is styled with text-primary (orange) and a hover underline so
   it stands out from the form without competing with the main Register button.

Why this approach:
  - No new backend work needed — the destination pages (/certificate-payment
    and /speaker-booking) already exist with their own API calls.
  - Keeps concerns separated: EventAddons.jsx is purely a navigation hub.
  - Works for people who missed the success-screen prompts or who want to
    book before/after registering.
