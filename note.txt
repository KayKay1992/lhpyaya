FEATURE: Event Description Redesign (RegisterEvent)
----------------------------------------------------
The plain white box showing the event description on the registration page
has been replaced with a modern, eye-catching gradient card.

What changed:
- The description is now displayed inside a bold orange-to-amber gradient
  card (rounded-3xl) with decorative circle accents behind it.
- A Sparkles icon badge + "ABOUT THIS EVENT" label sits above the text.
- Below the description, pill badges automatically show the event Date,
  Time, and Location pulled directly from the database — so the visitor
  gets all key facts at a glance without scrolling.
- Text is white on the gradient background for strong contrast and
  modern feel.

File: frontend/src/pages/RegisterEvent.jsx

FEATURE: Meet the Speakers Section (RegisterEvent)
----------------------------------------------------
A full "Meet the Speakers" section was added to the registration page,
appearing between the event description and the registration form.

What it does:
- Fetches speakers from the event API (/api/events/:id) and displays
  them as individual cards.
- Each speaker card shows:
    • Photo (or an orange avatar placeholder if no image uploaded)
    • Full Name (bold)
    • Role / Title / Bio (in primary orange colour)
    • A mic icon accent on the right
- "Register Now — Secure Your Spot!" buttons are interspersed
  automatically in the speakers list: one after every 2 speakers,
  and one final button at the bottom of the section.
- Clicking any "Register Now" button smoothly scrolls the page down
  to the registration form (using useRef + scrollIntoView).
- If no speakers have been added by the admin, the section is hidden
  entirely (no empty space shown).

Files changed:
- frontend/src/pages/RegisterEvent.jsx
  (added useRef, speakers state, scrollToForm helper, speakers JSX)

FEATURE: Speaker Management — Admin (EditEvent)
----------------------------------------------------
A "Manage Speakers" panel was added at the bottom of the Edit Event page
so the admin can add or remove speakers for any event.

What the admin can do:
- View all currently added speakers (photo, name, role).
- Delete any speaker with the red trash button (instant API call,
  no page reload needed).
- Add a new speaker using the form at the bottom of the panel:
    • Optional photo upload (image preview shown before saving)
    • Full Name (required)
    • Role / Title / Bio (optional, shown in orange on the public page)
  Clicking "Add Speaker" sends the data directly to the API.
  The speaker list updates immediately without refreshing the page.
- There is no limit — the admin can add as many speakers as needed.

Files changed:
- frontend/src/pages/EditEvent.jsx
  (added speakers state, speakerForm state, handleAddSpeaker,
   handleDeleteSpeaker, and the full Manage Speakers UI section)

FEATURE: Auto-redirect to Edit Page After Event Creation
----------------------------------------------------
Previously, after creating a new event the admin was sent straight to
the Dashboard with no opportunity to add speakers.

Now, after a successful event creation, the admin is automatically
redirected to the Edit Event page for that new event, where the
Manage Speakers panel is immediately available.

A toast message "Event created! Now add speakers below." confirms
the action and guides the admin.

File: frontend/src/pages/CreateEvent.jsx

----------------------------------------------------
BACKEND: Speakers API Endpoints
----------------------------------------------------
Two new API endpoints were added to support speaker management:

  POST   /api/events/:id/speakers
    — Adds a new speaker to the event.
    — Accepts multipart/form-data with fields: name, title, image (file).
    — Image is uploaded to Cloudinary (same storage as event banners).
    — Returns the full updated event object.
    — Protected: admin only.

  DELETE /api/events/:id/speakers/:speakerId
    — Removes a speaker by their subdocument _id.
    — Returns the full updated event object.
    — Protected: admin only.

Files changed:
- backend/models/Event.js
    (added speakers array: [{ name, title, image }])
- backend/controllers/eventController.js
    (added addSpeaker and deleteSpeaker functions)
- backend/routes/eventRoutes.js
    (registered the two new routes with protect + adminOnly middleware
     and Cloudinary upload middleware)
