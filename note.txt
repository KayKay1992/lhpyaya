---------------------------------------------------
1. AuthContext.jsx — How it works
---------------------------------------------------
React Context lets you share state across the whole app without passing props
through every component. AuthContext shares the logged-in user's info, loading
state, and login/logout functions.

STATE
  user    — holds the logged-in user object (name, email, role) or null if not logged in.
  loading — true while the app is checking whether the user is already logged in
            (prevents the app from flashing the login page on refresh).

useEffect ON MOUNT (auto-login on refresh)
  When the app first loads, it checks localStorage for a saved JWT token.
  - If no token exists: sets loading to false and does nothing.
  - If a token exists: calls GET /api/auth/me using the api instance (which
    automatically attaches the token in the Authorization header).
    - On success: sets the user state with the returned profile.
    - On failure (expired/invalid token): removes the bad token from
      localStorage and sets user to null.
  - Either way: sets loading to false when done.
  This means the user stays logged in across page refreshes.

login(token, userData)
  Called after a successful login or signup. Saves the JWT token to localStorage
  and sets the user state — triggering a re-render across the whole app.

logout()
  Removes the token from localStorage and sets user to null — instantly
  making the app treat the user as logged out.

useAuth() — custom hook
  Exported as a shortcut so any component can just write:
    const { user, loading, login, logout } = useAuth();
  instead of:
    const { user, loading, login, logout } = useContext(AuthContext);

---------------------------------------------------
2. main.jsx — AuthProvider wrapping
---------------------------------------------------
AuthProvider is wrapped around the entire App inside BrowserRouter:

  <BrowserRouter>
    <AuthProvider>
      <App />
    </AuthProvider>
  </BrowserRouter>

This makes the auth state (user, loading, login, logout) available to every
component in the app, including all pages and the ProtectedRoute.

---------------------------------------------------
3. ProtectedRoute.jsx — Updated
---------------------------------------------------
Previously used hardcoded placeholders (isAuthenticated = false).
Now it reads real state from AuthContext via the useAuth() hook:

  const { user, loading } = useAuth();

  - While loading is true: shows a loading message (prevents redirect flash).
  - If user is null (not logged in): redirects to /login, preserving the
    page the user was trying to reach in location.state.from.
  - If user exists: renders the protected children components.

This is the gatekeeper for all admin routes (/dashboard, /create-event, etc.).
