===================================================
FEATURE: Beautiful Loading Spinners Across All Pages
===================================================

User request:
  Replace all plain "Loading..." / "Submitting..." text with a beautiful
  spinner on a modern background — applied to every page in the app.

---

HOW IT WAS IMPLEMENTED
-----------------------

STEP 1 — Created a single reusable component file
  File: frontend/src/components/Spinner.jsx

  Three named exports were created in this one file so any page can
  import only what it needs:

  1. PageSpinner  — for when an ENTIRE page is loading
     - Renders a full-screen div with the same orange-50 gradient
       background used across the app (bg-linear-to-br from-orange-50
       via-white to-amber-50).
     - Contains an 80px ring made of two layered border divs:
         • Outer static ring: border-orange-100 (the "track")
         • Inner spinning arc: border-t-[#ff9324] + animate-spin
     - Inside the ring, a small white circle holds a 16px dot
       (bg-[#ff9324] + animate-pulse) that glows in sync.
     - Below the ring: a bold label (animate-pulse) + grey subtitle
       "Please wait a moment…"

  2. SectionSpinner  — for when only a DATA SECTION inside a page loads
     - Identical ring design but scaled down to 56px.
     - No page background — just centered vertically with py-24.
     - Used in tables and event grids where the navbar/header
       should still be visible while data loads.

  3. ButtonSpinner  — for SUBMIT BUTTONS while a form request is in-flight
     - Renders an inline <span> containing:
         • A 16px SVG circle spinner (animate-spin, currentColor so it
           inherits the button's text colour — white on orange buttons).
         • The label text beside it (e.g. "Submitting…", "Signing in…").
     - Keeps the button the same size so the layout doesn't jump.
     - Uses the classic "donut" SVG path:
         <circle opacity-25 /> for the track
         <path opacity-75 /> for the arc (the spinning part)

---

STEP 2 — Updated every page that had a loading state

  SECTION-LEVEL SPINNERS (SectionSpinner):
  These pages had a plain <div className="text-center py-20 text-gray-400">
  Loading...</div> block inside a conditional. Each was replaced with:
    <SectionSpinner label="Loading …" />

  Pages updated:
  • AdminDashboard.jsx     — was "Loading events..."
  • LandingPage.jsx        — was "Loading events..."
  • ViewRegistered.jsx     — was "Loading registrants..."
  • AdminCertificates.jsx  — was "Loading..."
  • AdminSpeakerBookings.jsx — was "Loading..."

  FULL-PAGE SPINNER (PageSpinner):
  EditEvent.jsx has a separate "fetching" state that runs before the page
  renders (it fetches the existing event data). It previously returned a
  bare min-h-screen centered div. Now it returns:
    <PageSpinner label="Loading event…" />

  BUTTON SPINNERS (ButtonSpinner):
  Every submit button previously had:
    {loading ? "Submitting..." : "Button Text"}
  This was replaced with:
    {loading ? <ButtonSpinner label="Submitting…" /> : "Button Text"}

  Pages updated:
  • RegisterEvent.jsx      — "Submitting…"
  • CertificatePayment.jsx — "Submitting…"
  • SpeakerBooking.jsx     — "Submitting…"
  • LoginPage.jsx          — "Signing in…"
  • SignupPage.jsx         — "Creating account…"
  • CreateEvent.jsx        — "Creating…"
  • EditEvent.jsx          — "Saving…"

---

WHY THIS APPROACH

  • Single source of truth — all spinner styles live in one file.
    To change the brand colour or animation, edit Spinner.jsx once.
  • No extra libraries — uses only Tailwind CSS utility classes and a
    small inline SVG. No react-spinners, no framer-motion dependency.
  • Three sizes for three contexts — full-page, section, and button
    each have the right visual weight for where they appear.
  • animate-spin is a standard Tailwind utility (1 full rotation per
    0.75 s linear). animate-pulse makes the inner dot breathe softly,
    giving the spinner a layered, modern feel.
  • The button spinner inherits currentColor so it automatically turns
    white on the orange buttons without any extra class.
