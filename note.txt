---------------------------------------------------
1. LANDINGPAGE.JSX
---------------------------------------------------
PURPOSE
  The public-facing home page. Anyone can view it without logging in.
  It fetches all events from the backend and displays them as cards.

STATE
  events  — array of event objects returned from the API.
  loading — true while the API call is in progress.

DATA FETCHING — useEffect
  On first render, calls GET /api/events via the api instance.
  - On success: stores the events array in state.
  - On error: sets events to an empty array (no crash).
  - Always: sets loading to false when done.

NAVBAR
  - Shows the app brand/logo on the left.
  - On the right, checks the AuthContext user:
    - Not logged in → "Sign In" button → /login
    - Logged in as admin → "Dashboard" button → /dashboard
    - Logged in as regular user → shows "Hello, <name>"
  This is sticky (stays at top while scrolling) with a frosted glass effect.

HERO SECTION
  Static marketing text with a badge, heading, and tagline.

EVENTS GRID
  Three states:
  1. Loading  — shows a simple "Loading events..." message.
  2. No events — shows a friendly empty state with an icon.
  3. Has events — renders a responsive grid (1 → 2 → 3 columns).

EVENT CARD
  Each card shows:
  - Event image (if one was uploaded) or a gradient placeholder.
  - Title, description (clamped to 2 lines).
  - Date (formatted with toLocaleDateString), time, location — each with an icon.
  - "Register Now" button.

  Clicking "Register Now" navigates to:
    /register?eventId=<id>&title=<encodedTitle>
  The eventId and title are passed as URL search params so the registration
  page knows which event to register for without a separate API call.

formatDate helper
  Converts the ISO date string from MongoDB into a readable format:
  e.g. "Wed, May 1, 2026"

---------------------------------------------------
2. REGISTEREVENT.JSX
---------------------------------------------------
PURPOSE
  A public form where anyone (no account needed) can register for a specific event.

URL PARAMS — useSearchParams
  Reads eventId and title from the URL query string:
    /register?eventId=abc123&title=Tech%20Summit
  This is how the page knows which event to submit the registration for.

STATE
  form    — { name, email, phone } controlled form fields.
  loading — true while the API call is in progress.
  success — flips to true after a successful registration to show the success screen.

FLOW
  1. User fills in name and email (phone is optional).
  2. On submit, validates that eventId exists in the URL.
  3. Calls POST /api/registrations/:eventId with { name, email, phone }.
  4. On success: sets success = true → shows the success screen.
  5. On error (e.g. already registered): shows a toast error message.

SUCCESS SCREEN
  Instead of navigating away and losing context, the component swaps its UI
  to a full success card with a green checkmark, a confirmation message showing
  the event name, and a "Back to Events" button that navigates to /.

BACK LINK
  An arrow link at the top navigates back to the landing page without
  losing form context (uses Link, not navigate, so it is accessible).

UI DESIGN
  - Same warm gradient background and rounded-3xl card as the auth pages.
  - Brand-coloured focus rings, bold submit button with orange shadow.
  - Event title displayed in orange below the heading so users always know
    which event they are registering for.
