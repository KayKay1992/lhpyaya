FEATURE 1: "WHO TOLD YOU ABOUT THIS EVENT?" — REFERRAL FIELD
====================================================================

PURPOSE:
  Captures how each participant heard about the event so the
  organiser can track which outreach channels are most effective.

BACKEND CHANGES:

  1. backend/models/Registration.js
     - Added a new optional field to the schema:
         referral: { type: String, trim: true, default: "" }
     - It is optional (not required) so existing registrations
       without the field are unaffected.

  2. backend/controllers/registrationController.js  — registerForEvent
     - Destructures `referral` from req.body alongside name, email,
       and phone.
     - Passes `referral: referral || ""` into Registration.create()
       so the value is saved to MongoDB.

FRONTEND CHANGES:

  3. frontend/src/pages/RegisterEvent.jsx
     - Form state updated from { name, email, phone } to
       { name, email, phone, referral: "" }.
     - A new optional input field is rendered after the Phone field:
         Label:       "Who told you about this event?"
         Placeholder: "e.g. A friend, Social media, Poster,
                       Church announcement..."
         Type:        text (free-form, not required)
     - The field value is submitted with the rest of the form.

  4. frontend/src/pages/ViewRegistered.jsx  (admin view)
     - Added MessageSquare to the lucide-react icon imports.
     - Desktop table grid expanded from 6 columns to 7 columns:
         [Name | Email | Phone | Registered | Referral | Attended | Delete]
     - "Referral" column header added with a MessageSquare icon.
     - Each registrant row now shows the referral answer, or an
       em-dash (—) if the participant left it blank.
     - Mobile rows are unchanged (already space-constrained).

FEATURE 2: LIVE ATTENDANCE COUNT
====================================================================

PURPOSE:
  Gives the admin an at-a-glance summary of how many registered
  participants actually showed up, without manual counting.

BACKEND:
  No backend changes needed. The `attended` boolean already exists
  on every Registration document and is returned in the existing
  GET /api/registrations/:eventId response.

FRONTEND CHANGES:

  frontend/src/pages/ViewRegistered.jsx
  - The header stat area previously showed one orange pill:
        "X registered"
  - It now shows TWO pills side by side:
        🟠  "X registered"   (orange — total sign-ups)
        🟢  "X attended"     (green  — total marked as attended)
  - The attended count is computed inline:
        registrants.filter((r) => r.attended).length
  - Because registrants state is updated immediately every time
    the admin clicks "Mark" / "Attended" (via the existing
    handleToggleAttend function), the green count updates in
    real time without any page reload.

FEATURE 3: CERTIFICATE & SPEAKER PROMPTS — REDESIGNED & MOVED
====================================================================

PURPOSE:
  The previous prompts were small text lines with underlined links
  that were easy to miss. They are now bold, colourful action cards
  that are impossible to overlook. They have also been repositioned
  so participants see the event details first and then are presented
  with the add-on offers.

PREVIOUS ORDER on the success screen:
  1. ✅ "You're registered!" heading + message
  2. Certificate text prompt  (amber, small text)
  3. Speaker text prompt      (blue, small text)
  4. Event details card       (date / time / location)
  5. "Back to Events" button

NEW ORDER on the success screen:
  1. ✅ "You're registered!" heading + message
  2. Event details card       (date / time / location)  ← moved up
  3. Certificate action card  (amber, full card)        ← redesigned
  4. Speaker action card      (blue, full card)         ← redesigned
  5. "Back to Events" button

CERTIFICATE CARD (amber theme):
  - Border:      2px solid amber-300
  - Background:  amber-50
  - Icon badge:  9×9 amber-400 rounded square with a white Award icon

  - Button:      Full-width solid amber-400 button
                 Label: "Yes, Get My Certificate →"

SPEAKER BOOKING CARD (blue theme):

HOW EVERYTHING FITS TOGETHER

     participant heard about the event.
