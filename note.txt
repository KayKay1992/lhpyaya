  SPEAKER SESSION BOOKING FEATURE — IMPLEMENTATION NOTES
  Date: May 20, 2026
====================================================================

OVERVIEW
--------
A full "Book a One-on-One Session with the Keynote Speaker" flow was
added to the LHP-YAYA app. This is separate from the Certificate of
Participation feature and sits alongside it on the registration
success screen. It covers a participant-facing payment page, the
success screen prompt, and an admin management page for viewing all
bookings. Both backend and frontend were updated.

====================================================================
BACKEND CHANGES
====================================================================

1. NEW FILE: backend/models/SpeakerBooking.js
   - Mongoose model that stores one booking record per participant.
   - Fields:
       • event          — ref to Event model (ObjectId), required
       • name           — participant's full name, required, trimmed
       • phone          — participant's phone number, required, trimmed
       • paymentEvidence — Cloudinary URL of the uploaded payment proof
       • status         — "pending" | "approved" | "rejected"
                          (default: "pending")
   - Timestamps (createdAt, updatedAt) are auto-added by Mongoose.

2. NEW FILE: backend/controllers/speakerBookingController.js
   Two exported functions:

   a) submitBooking  — handles POST /api/speaker-bookings  (PUBLIC)
      • Validates that eventId, name, and phone are all present.
      • Requires a file to be uploaded (paymentEvidence); returns 400
        if missing.
      • Validates phone number format: 7–15 digits, optional leading +,
        allowing spaces, dashes, dots, and parentheses as separators.
      • Saves a new SpeakerBooking document to MongoDB.
      • Returns 201 with the created document on success.

   b) getAllBookings  — handles GET /api/speaker-bookings  (ADMIN ONLY)
      • Fetches all SpeakerBooking documents from the database.
      • Populates the event field with title and date for display.
      • Sorts results by newest first (createdAt: -1).
      • Returns the array as JSON.

3. NEW FILE: backend/routes/speakerBookingRoutes.js
   - POST /   → certificateUploadMiddleware.single("paymentEvidence")
                → submitBooking
                (reuses the existing Cloudinary upload middleware that
                 already supports JPG/PNG/WEBP/PDF up to 10 MB, uploaded
                 to the "lhpyaya-payment-evidence" folder on Cloudinary)
   - GET  /   → protect → adminOnly → getAllBookings

4. UPDATED: backend/server.js
   - Registered the new route:
       app.use("/api/speaker-bookings",
               require("./routes/speakerBookingRoutes"))

HOW THE FLOW WORKS END-TO-END
====================================================================

PARTICIPANT SIDE:
  1. Participant fills in the registration form and submits.
  2. Success screen appears — two prompt cards are shown:
       • Amber card  → Certificate of Participation (₦2,000)
       • Blue card   → One-on-one with Keynote Speaker (₦20,000)
  3. Participant clicks "Click Here!" on the blue card.
  4. Lands on /speaker-booking with their name pre-filled.
  5. Reads the payment details and makes a ₦20,000 bank transfer to:
         RCCG GLORIOUS YOUTH FELLOWSHIP
         FIDELITY BANK — 5210040973
  6. Enters their name, phone number, and uploads payment screenshot.
  7. Clicks "Submit Booking Request" — sees confirmation screen.

ADMIN SIDE:
  1. Admin logs in → Admin Dashboard.
  2. Clicks "Speaker Sessions" button in the header.
  3. Sees a full table of every participant who booked a session,
     showing their Name, Phone Number, and Event.
  4. Clicks "View Receipt" on any row to verify the participant's
     payment evidence before confirming their session slot.
