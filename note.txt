  CERTIFICATE OF PARTICIPATION FEATURE — IMPLEMENTATION NOTES
  Date: May 20, 2026
====================================================================

OVERVIEW
--------
A full Certificate of Participation flow was added to the LHP-YAYA
app, covering the participant-facing payment page, the success screen
prompt, and an admin management page. Both frontend and backend were
updated.

====================================================================
BACKEND CHANGES
====================================================================

1. NEW FILE: backend/models/CertificateRequest.js
   - Mongoose model that stores one certificate request per participant.
   - Fields:
       • event        — ref to Event model (ObjectId)
       • participantName — the participant's real/registered name
       • certificateName — exactly how they want the name on the cert
       • email        — participant's email
       • paymentEvidence — Cloudinary URL of the uploaded proof of payment
       • status       — "pending" | "approved" | "rejected" (default: pending)
   - Timestamps (createdAt, updatedAt) are auto-added.

2. NEW FILE: backend/middleware/certificateUploadMiddleware.js
   - Separate Multer + Cloudinary storage config for payment evidence.
   - Saves files to the "lhpyaya-payment-evidence" folder on Cloudinary.
   - Accepts: JPG, PNG, WEBP images and PDF files.
   - File size limit: 10 MB.

3. NEW FILE: backend/controllers/certificateController.js
   - submitRequest  (POST /api/certificates) — PUBLIC
       • Validates all required fields and email format.
       • Requires a file upload (paymentEvidence).
       • Saves a new CertificateRequest document to the database.
   - getAllRequests (GET  /api/certificates) — ADMIN ONLY
       • Returns all certificate requests sorted newest first.
       • Populates event title and date for display.

4. NEW FILE: backend/routes/certificateRoutes.js
   - POST /   → certificateUpload.single("paymentEvidence") → submitRequest
   - GET  /   → protect → adminOnly → getAllRequests

5. UPDATED: backend/server.js
   - Registered the new route:
       app.use("/api/certificates", require("./routes/certificateRoutes"))

====================================================================
HOW THE FLOW WORKS END-TO-END
====================================================================

PARTICIPANT SIDE:
  1. Participant registers for an event → sees the success screen.
  2. Sees the certificate prompt → clicks "Click Here!".
  3. Lands on /certificate-payment with their name pre-filled.
  4. Reads the payment details, makes a bank transfer of ₦2,000.
  5. Fills in their certificate name and email, uploads payment screenshot.
  6. Submits → sees confirmation screen.

ADMIN SIDE:
  1. Admin logs in → clicks "Certificates" on the dashboard.
  2. Sees a table of all participants who requested a certificate.
  3. For each entry:
       – Clicks "View Receipt" to verify the payment evidence.
       – Clicks "Generate Certificate" to produce and print the certificate
         with the participant's preferred name.
