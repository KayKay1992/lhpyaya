---------------------------------------------------
1. ADMINDASHBOARD.JSX
---------------------------------------------------
PURPOSE
  The main admin control panel. Lists all events in a card grid with action buttons.

STATE
  events     — array of all events fetched from the API.
  loading    — true while fetching.
  deletingId — tracks which event is being deleted to disable its button.

NAVBAR
  Shows LHP-YAYA logos, an "Admin" badge, the logged-in admin's name,
  a "View Site" link to the landing page, and a Sign Out button that
  calls logout() and redirects to /login.

FETCH ON MOUNT
  Calls GET /api/events via fetchEvents(). Extracted as a named function
  (not inline in useEffect) so it can be called again if needed.

DELETE FLOW
  1. window.confirm() asks for confirmation before proceeding.
  2. Sets deletingId to the event's id (disables that card's delete button).
  3. Calls DELETE /api/events/:id.
  4. On success: filters the deleted event out of local state immediately —
     no need to re-fetch the whole list.
  5. Resets deletingId in the finally block.

CARD ACTIONS (per event)
  - "Registrants" → navigates to /view-registered?eventId=...&title=...
  - "Edit"        → navigates to /edit-event/:id
  - "Delete"      → triggers the delete flow above

---------------------------------------------------
2. CREATEEVENT.JSX
---------------------------------------------------
PURPOSE
  Admin form for creating a new event, including an optional banner image.

STATE
  form    — { title, description, date, time, location } controlled fields.
  image   — the raw File object selected by the admin.
  preview — a local object URL for showing the image preview before upload.
  loading — true while the API call is in progress.

IMAGE UPLOAD UI
  A dashed-border clickable area wraps a hidden <input type="file">.
  When a file is selected, URL.createObjectURL() creates a temporary
  browser-side URL to show a live preview inside the upload area.

SUBMIT FLOW
  Uses FormData (not JSON) because the request includes a file:
  1. Appends all text fields to FormData with Object.entries(form).forEach().
  2. Appends the image file if one was selected.
  3. Sends POST /api/events with Content-Type: multipart/form-data.
     (The api instance default is application/json, so this is overridden here.)
  4. On success: toast + navigate back to /dashboard.

---------------------------------------------------
3. EDITEVENT.JSX
---------------------------------------------------
PURPOSE
  Same layout as CreateEvent but pre-fills fields with the existing event's data.

KEY DIFFERENCE — useEffect on mount
  Reads the event id from useParams() (URL: /edit-event/:id).
  Calls GET /api/events/:id to fetch the current data and populate the form.
  
  Date handling: MongoDB stores dates as ISO strings (e.g. "2026-05-01T00:00:00Z").
  .slice(0, 10) trims it to "2026-05-01" which is the format <input type="date"> needs.

  Existing image: if the event has an image, sets the preview to its full URL
  (http://localhost:5000/<path>) so the admin can see the current banner.

SUBMIT FLOW
  Same as CreateEvent but calls PUT /api/events/:id. Only sends a new image
  file if the admin picked one — otherwise the existing image stays unchanged
  (handled by the backend's "update only what was sent" pattern).

fetching state
  Shows a simple "Loading event..." screen while the initial GET is in flight,
  preventing the form from rendering with empty values.

---------------------------------------------------
4. VIEWREGISTERED.JSX
---------------------------------------------------
PURPOSE
  Shows the admin a table of everyone who registered for a specific event.

URL PARAMS — useSearchParams
  Reads eventId and title from the URL query string:
    /view-registered?eventId=abc123&title=Tech%20Summit

DATA FETCHING
  Calls GET /api/registrations/:eventId on mount.
  The response contains: { event (title), total, registrants[] }.
  Sets both eventTitle and registrants from the response.

TABLE LAYOUT
  A 4-column grid (CSS grid, not <table>) for clean alignment:
    Name | Email | Phone | Registered (date)
  
  - Alternating row background (white / gray-50) for readability.
  - Phone shows an em dash (—) if the field is empty.
  - Registration date formatted as "May 1, 2026".
  - A badge in the header shows the total count.

EMPTY STATE
  If no one has registered, shows a friendly icon and message.

---------------------------------------------------
SHARED DESIGN PATTERNS
---------------------------------------------------
  - All admin pages use the same warm gradient background.
  - Back arrow links use Link (not navigate) for proper accessibility.
  - All forms disable the submit button and show loading text during API calls.
  - Errors are shown via react-hot-toast.
  - All admin routes are protected by ProtectedRoute in App.jsx.
